# Spec: Patch 3.2.0 — Protocol Compliance and Concept-Agent Path Fix

## Date
2026-04-10

## Status
Approved

## Overview

Three fixes to the claude-professor whiteboard plugin. Implemented as three separate commits on a feature branch, squashed to main on merge.

**Root problem:** The LLM running whiteboard optimizes for forward momentum — skipping concept-agent calls, submitting non-registry candidates, and building on concepts without verifying user knowledge. The plugin's entire purpose is to pause that momentum. These fixes make compliance structural rather than relying on prompt text alone.

**Fix order (Option B — code-first):**
1. `session.js` gate command + tests
2. `concept-agent.md` path bug fix
3. `skills/whiteboard/SKILL.md` enforcement

## Commit 1: session.js Gate Command

### What changes
`scripts/session.js` — new `gate` subcommand added alongside `create`, `load`, `update`, `add-concept`, `clear`.

### Interface
```bash
node session.js gate \
  --require concepts \
  --session-dir docs/professor/
```

### Behaviour

| Condition | Exit code | stdout |
|---|---|---|
| `concepts_checked.length > 0` | 0 | `{ "gate": "open" }` |
| `concepts_checked.length === 0` | 1 | `{ "gate": "blocked", "reason": "concepts_checked is empty — run concept-agent before proceeding" }` |
| No active session file | 0 | `{ "gate": "open", "warning": "no active session found" }` |
| Unknown `--require` value | 1 | usage error message |

### Scope
Gate fires at two phase boundaries only:
- Before Phase 1.6 (requirement discussion)
- Before Phase 2.1 (design proposals)

Lazy inline concept checks during discussion are **not** gated in this patch — deferred to 3.3.0.

### Tests
Added to `scripts/test/session.test.js`:

1. **Gate opens when `concepts_checked` is non-empty** — create session, add a concept, call gate, assert exit 0 and `{ "gate": "open" }`
2. **Gate blocks when `concepts_checked` is empty** — create session with no concepts, call gate, assert exit 1 and `"gate": "blocked"`
3. **Gate opens with warning when no session exists** — call gate against empty dir, assert exit 0 and `"warning"` field present
4. **Gate exits non-zero on unknown `--require` value** — call gate with `--require unknown`, assert exit 1

## Commit 2: concept-agent Path Bug Fix

### What changes
`agents/concept-agent.md` — Compute Status step only.

### The bug
Compute Status instructs the LLM to check:
```
~/.claude/professor/concepts/{concept_id}.md
```
Profiles are stored at:
```
~/.claude/professor/concepts/{domain}/{concept_id}.md
```
Result: every concept with an existing profile is reported as `new`.

### The fix
Replace the path in the Compute Status instructions:

**Before:**
> "Check if a user profile file exists for the concept in `~/.claude/professor/concepts/`"

**After:**
> "Check if a user profile file exists at `~/.claude/professor/concepts/{domain}/{concept_id}.md` — use the domain resolved in the previous step."

### Error model

| Condition | Handling |
|---|---|
| File exists | Read it, run `lookup.js status`, compute FSRS action |
| File missing (domain dir exists) | Status `new` |
| Domain dir missing | Status `new` — treat as never written, same as missing file |
| Domain not in 18-domain taxonomy | Typed error — record in `ambiguous` with `reason: "invalid_domain"` |

### Write path (resolve-or-create mode)
Before writing a new concept file, add:
> "Ensure the domain directory exists: `mkdir -p {profile_dir}/{domain}/` before writing the file."

### Circuit breaker
No change to existing systemic/input-specific classification from v3.1.0. "File not found" remains input-specific (HALF-OPEN). Script logic errors remain systemic (OPEN).

### Verification
Smoke test: run `lookup.js status --concepts circuit_breaker` after fix — must return `skip` (R ≈ 1.0), not `new`.

## Commit 3: whiteboard SKILL.md Enforcement

### What changes
`skills/whiteboard/SKILL.md` — two sections updated.

### Fix 1: Phase 1.4 — L1-only candidate filter

Add after the candidate identification instruction:

> "Only submit candidates that are resolvable L1 seed concepts from the registry. Do not submit candidates with no registry entry — terms like `spaced_repetition`, `retrievability`, `knowledge_graph` are not in the seed registry and cannot be resolved in `resolve-only` mode. Non-registry terms belong in Phase 3 LLD via `resolve-or-create` mode."

### Fix 2: Phase 1.6 — Gate call before requirement discussion

Add at the start of Phase 1.6, before any discussion begins:

```bash
node ${CLAUDE_PLUGIN_ROOT}/scripts/session.js gate \
  --require concepts \
  --session-dir docs/professor/
```

> "If this exits non-zero, STOP. Do not begin requirement discussion until concept-agent has been called and at least one concept is recorded in session state."

### Fix 2: Phase 2.1 — Gate call before design proposals

Add at the start of Phase 2.1, before proposing design options:

```bash
node ${CLAUDE_PLUGIN_ROOT}/scripts/session.js gate \
  --require concepts \
  --session-dir docs/professor/
```

> "If this exits non-zero, STOP. Do not propose design options until concept-agent has been called and concepts are recorded."
